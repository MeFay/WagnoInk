Visão Geral do Projeto

O wagno.ink é um portfolio digital profissional criado para o tatuador Wagno, com base em Porto, Portugal. O objetivo do website é apresentar o trabalho artístico, atrair novos clientes, facilitar marcações e reforçar a presença online nas pesquisas do Google e nas redes sociais.

Design Visual — Identidade e Intenção

Cada decisão de design foi tomada com intenção clara: criar uma presença online que seja imediatamente reconhecível como obra do Wagno — com a mesma precisão e identidade que existe no trabalho dele.

Paleta de Cores

A paleta foi construída à volta de três pilares: o preto como base de sofisticação, o vermelho como energia e apelo à ação, e o âmbar/dourado como prestígio e conquista.

Porquê esta combinação? O preto foi escolhido porque é o fundo natural da tinta — é onde o trabalho do Wagno existe. O vermelho é a cor que mais se destaca nesse fundo escuro, funcionando como o gesto direto e decisivo de quem tatua. O âmbar/dourado entra como a recompensa: o brilho da qualidade, os prémios, o reconhecimento. Em conjunto, a paleta comunica que este é um artista que leva o trabalho a sério, sem precisar de gritar por atenção.

Funcionalidades

  • Layout Masonry — colunas de alturas variadas, semelhante ao Pinterest. As imagens aproveitam melhor o espaço e criam um visual dinâmico.
  • Categorias — Realismo, Blackwork, Fine Line. Permite a clientes filtrar pelo estilo que procuram.
  • Lightbox — ao clicar numa imagem, abre em ecrã completo com navegação. Botões ← →, teclas de teclado e tecla Esc para fechar.
  • Botão Baralhar — aleatoriza a ordem da galeria. Clientes que visitam mais do que uma vez veem sempre uma disposição diferente.
  • Frames flutuantes — no cabeçalho da página, 3 fotografias flutuam animadas — visual imersivo que convida à exploração.
  • Hover com zoom — ao passar o rato, a imagem faz zoom subtil e aparece uma sobreposição com o título. Feedback visual polido.
  • Lazy loading — as imagens só carregam quando visíveis no ecrã. Página mais rápida, especialmente em mobile.
  • Back button no mobile — ao abrir o lightbox no telemóvel, o botão de retrocesso fecha a imagem em vez de sair do site.

SEO — Visibilidade no Google

Uma série de otimizações técnicas foram implementadas para garantir que o Google indexa e apresenta o website nos resultados de pesquisa de forma correta e atrativa.

O que foi implementado

  • robots.txt — instrui os motores de pesquisa a indexar todas as páginas do website.
  • sitemap.xml — mapa completo do site com prioridades. Homepage (prioridade 1.0) → Galeria (0.9) → Sobre (0.8) → Info (0.7) → Blog (0.6).
  • Títulos dinâmicos — cada página tem o seu próprio título na aba do browser, otimizado para pesquisa.
  • Meta descrição — texto de apresentação nos resultados do Google, escrito para atrair cliques.
  • Open Graph — quando o link é partilhado no WhatsApp, Facebook ou LinkedIn, aparece uma pré-visualização bonita com imagem, título e descrição.
  • JSON-LD / Schema.org — dados estruturados que dizem ao Google que este é um negócio local do tipo "Salão de Tatuagens", com morada, telefone, horários e métodos de pagamento.

Performance e Velocidade

A velocidade do website afeta diretamente a experiência do utilizador e o posicionamento no Google. Foram tomadas diversas decisões técnicas para garantir carregamento rápido.

  • Vite como bundler — ferramenta moderna que produz ficheiros otimizados para produção. Carregamento inicial muito mais rápido que alternativas.
  • Code splitting — cada página carrega de forma independente. O visitante não precisa de descarregar o código de todas as páginas de uma vez.
  • Imagens WebP — formato moderno, tipicamente 25–35% mais leve que JPG com a mesma qualidade visual.
  • Lazy loading de imagens — as imagens da galeria só carregam quando visíveis no ecrã. A página fica pronta mais rápido e não desperdiça dados do utilizador.
  • Fontes com display:swap — o texto é visível imediatamente, mesmo antes da fonte Geist terminar de carregar.
  • Preload de recursos críticos — o Vite pré-carrega os ficheiros mais importantes antes de serem necessários.

Regra do Google: Páginas que carregam em menos de 3 segundos têm taxas de rejeição significativamente mais baixas e melhor posicionamento orgânico.

Animações e Experiência de Utilizador

As animações foram escolhidas com critério: subtis o suficiente para não distrair, impactantes o suficiente para elevar a qualidade percebida do website.

Princípios adotados

  • Reduced Motion — o sistema respeita a preferência do sistema operativo do utilizador. Quem tem animações desativadas (por questões de acessibilidade ou preferência) vê a página sem animações.
  • Suavidade consistente — todas as transições usam curvas de aceleração (easing) naturais. Nada é abrupto ou mecânico.
  • Stagger — na galeria e em listas, os elementos entram escalonados (um a um), criando um efeito de "cascata" elegante.

Dispositivos e Responsividade

Mais de 60% do tráfego web vem de dispositivos móveis. O website foi desenvolvido com abordagem mobile-first — a experiência em telemóvel foi uma prioridade, não um afterthought.

DispositivoLayoutCaracterísticas específicas
Desktop ≥1024px Navbar no topo, sidebar na página Sobre, galeria 3 colunas Experiência completa com todos os detalhes visuais
Tablet 768–1023px Layout adaptado, galeria 2–3 colunas Navigation pill, elementos redimensionados
Mobile <768px Bottom pill navigation, galeria 2 colunas, layout empilhado Back button integrado no lightbox, toque otimizado

Funcionalidades exclusivas para mobile

  • Bottom Pill Navigation — barra de navegação fixa na parte inferior, com ícones e indicador da página ativa. Mais ergonómica que menus no topo.
  • Back button integrado — ao abrir uma foto em fullscreen, o botão de retroceder do telemóvel fecha a imagem em vez de sair do website.
  • Prevenção de long-press — desativado o menu contextual ao manter o dedo nas imagens (comportamento padrão iOS), evitando interações acidentais.
  • Touch optimizado — todos os botões e áreas clicáveis têm tamanhos adequados para interação com o dedo.

Acessibilidade

O website foi desenvolvido com boas práticas de acessibilidade, tornando-o utilizável por um maior leque de pessoas e melhor avaliado por motores de pesquisa.

  • Labels ARIA — todos os botões e ícones têm descrições para leitores de ecrã (utilizadores com deficiência visual).
  • Navegação por teclado — galeria (teclas ←/→/Esc), menus dropdown e formulários são totalmente navegáveis sem rato.
  • Contraste adequado — branco suave sobre fundo escuro garante legibilidade confortável.
  • Alt text nas imagens — descrições textuais em todas as imagens para leitores de ecrã e SEO.
  • Reduced motion — animações desativadas automaticamente para utilizadores com preferências de acessibilidade.
  • HTML semântico — estrutura correta com <nav>, <main>, <footer>, <section> — melhor para leitores de ecrã e SEO.
